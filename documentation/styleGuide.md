# Handbook Style Guide

This is a lightweight standard for personal coaching manuals and reference manuals in this repository.

## Purpose

The documents are practical working manuals, not formal publications. They should be easy to read, easy to update and consistent enough that content can be copied into LibreOffice with minimal reformatting.

## Core Workflow

1. Draft content in Markdown or plain text.
2. Paste into the LibreOffice document.
3. Use the shared template styles.
4. Avoid repeated manual style corrections.

If a pasted heading or paragraph regularly needs manual restyling, update the template rather than accepting the workflow problem.

## Standard LibreOffice Styles

Use built-in styles wherever possible.

| Purpose | LibreOffice Style |
|---|---|
| Chapter title | Heading 1 |
| Main section | Heading 2 |
| Subsection | Heading 3 |
| Minor heading | Heading 4 |
| Normal text | Text Body |
| Bullet lists | List 1 |
| Captions | Caption |
| Quotes | Quote |

## Custom Styles

Custom styles should be reserved for repeated document elements that do not map cleanly to built-in styles.

Recommended custom styles:

- Coaching Point
- Common Mistake
- Practice Drill
- Remember
- Analysis
- Example
- Diagram Caption
- Screenshot Caption

## Heading Rules

- Heading 1 starts a new chapter and should begin on a new page.
- Heading 2 is for major sections within a chapter.
- Heading 3 is for sub-sections.
- Heading 4 is for short named blocks or minor sections.
- Avoid custom heading styles such as `Chapter Title` or `Section Title`; instead, style `Heading 1`, `Heading 2`, and so on directly in the template.

## Writing Pattern

Most chapters should answer these questions in order:

1. What is it?
2. Why does it matter?
3. How do I do it?
4. What mistakes should I avoid?
5. How do I practise it?
6. When do I use it?
7. What should I remember?

## Template Principle

Improve the template, not the workflow.
